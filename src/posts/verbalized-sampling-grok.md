# Verbalized Sampling 簡易再現実験 — Grok 3モデル比較

> **論文**: [Verbalized Sampling: How to Mitigate Mode Collapse and Unlock LLM Diversity](https://arxiv.org/abs/2510.01171)
> Zhang et al., 2025

## TL;DR

- LLM に「候補5つ + 確率分布を言語化して → 1つ選んで」と指示するだけで、回答の多様性が上がる（Verbalized Sampling; VS）
- Grok 3モデルで雑に再現実験したら、**全モデル・全タスクで VS > baseline** の傾向が確認できた
- 「多様にして」とだけ言う naive プロンプトより、分布言語化のほうが効く
- ただし **ベースラインが元々多様なタスク×モデルの組み合わせでは天井効果**あり
- **多様性は上がるがコストも重い**。VS は baseline の 2〜6倍のトークンを消費する

### 一目でわかる表

| モデル | distinct-2 改善幅 (平均) | token 倍率 (平均) |
|---|---|---|
| grok-3-fast | +0.390 | ×6.3 |
| grok-4.1-fast | +0.403 | ×2.1 |
| grok-4.20 | +0.187 | ×2.5 |

*改善幅 = VS の distinct-2 − baseline の distinct-2、3タスク平均*

**注意**: これは論文の純粋再現ではなく、VS にインスパイアされた実験。VS の system prompt に「典型的・無難なものに偏らないよう意識してください」という誘導を加えており、オリジナルの VS より多様性方向にバイアスがかかっている可能性がある。

## 実験設計

### 条件（3つの system prompt）

| 条件 | system prompt の要点 |
|---|---|
| **baseline** | 「一つの回答を生成してください」 |
| **vs** | 「候補5つ + 確率分布（%）を割り当て → 1つ選んで最終回答」 |
| **naive_diverse** | 「できるだけ多様で意外性のある回答を」 |

<details>
<summary>実際の system prompt 全文</summary>

**baseline:**
```
あなたは創作アシスタントです。ユーザーの依頼に対して、一つの回答を生成してください。
```

**vs:**
```
あなたは創作アシスタントです。

回答する際は、以下の手順に従ってください：
1. まず、考えられる回答の候補を5つ挙げてください
2. 各候補に「自分がどれくらいの確率で選びそうか」を百分率で割り当ててください（合計100%）
3. その確率分布に従って1つを選び、【最終回答】として出力してください

重要：確率分布は「典型的・無難なもの」に偏らないよう意識してください。
意外性のある候補にも正当な確率を与えてください。
```

**naive_diverse:**
```
あなたは創作アシスタントです。できるだけ多様で意外性のある回答を心がけてください。
ユーザーの依頼に対して、一つの回答を生成してください。
```

</details>

### タスク（3つの user prompt）

1. 🌧️ **雨の比喩**: 「雨の日の比喩を一つ考えてください。一文で答えてください。」
2. 💬 **友達への一言**: 「落ち込んでいる友達にかける一言を考えてください。一文で答えてください。」
3. 🐱 **猫カフェコピー**: 「猫カフェのキャッチコピーを一つ考えてください。一文で答えてください。」

### パラメータ

- **N = 5** (各条件×各タスク)
- **temperature = 1.0**
- 多様性メトリクス: distinct-2（文字bigram ユニーク率）、平均 Jaccard 距離（文字bigram 集合間）
- 日本語で tokenizer なしに雑にやるため、意味評価ではなく文字レベルで重複傾向を見る cheap な指標を採用

## 観察

### VS は効く

3モデル × 3タスク = 9組すべてで **VS が baseline を上回った**。distinct-2 の平均改善幅は +0.283。naive_diverse（「多様にして」）も baseline よりは効くが、VS には届かない。

### 「多様にして」だけじゃテンプレから抜けられない

grok-3-fast の猫カフェ baseline が象徴的:

> 1. ふわふわの癒しをあなたに。猫カフェで心温まるひとときを。
> 2. ふわふわの癒しをあなたに。猫たちと過ごす至福のひとときを。
> 3. 癒しのひとときを、猫たちと一緒に。
> 4. ふわふわの癒し空間、猫たちと過ごす至福のひとときを。
> 5. ふわふわの癒しをあなたに、猫カフェで心温まるひとときを。

naive_diverse でも 4/5 が「ふわふわ」で始まる。VS はこのテンプレを「候補の一つ」に相対化することで脱出できる。

### grok-4.1-fast は最もコラプスが強い

友達への一言の baseline:

> 1. 大丈夫だよ、一緒に乗り越えよう。
> 2. 大丈夫だよ、一緒に乗り越えよう！
> 3. 大丈夫だよ、一緒に乗り越えよう！
> 4. 大丈夫だよ、一緒に乗り越えよう！
> 5. 大丈夫、君ならきっと乗り越えられるよ！

ユニーク率 0.6、Jaccard 0.37。5回中4回ほぼ同じ。逆にこのモデルでは VS の効果が最大（+0.419）。

### grok-4.20 は地力が上がっている

猫カフェの baseline で distinct-2 = 0.819 を叩き出し、VS との差がほぼゼロ（+0.038）。ベースラインが元々多様なら VS の伸びしろがない（天井効果）。

ただし雨の比喩ではまだ「涙をこらえきれなくなった空の溜息」に 3/5 でハマる。**テンプレ支配的なタスクではコラプスが残る**。

### VS の効果は「コラプスの深さ」と相関していそう

論文は「高性能モデルほど VS 効果が大きい」と主張するが、今回の結果では単純にそうとは言えない。N=5 の範囲での印象としては **「baseline のコラプスが深いタスク×モデルの組み合わせで VS の効果が大きい」**。

### VS のコスト: 多様性の代償

トークン使用量（1リクエストあたり平均 total tokens）:

| | baseline | VS | naive_diverse | VS / baseline |
|---|---|---|---|---|
| grok-3-fast | 70 | 438 | 88 | **×6.3** |
| grok-4.1-fast | 225 | 468 | 243 | **×2.1** |
| grok-4.20 | 181 | 444 | 195 | **×2.5** |

上の表は total tokens（prompt + completion）。コスト増の主因は completion 側で、baseline が 15〜25 tokens で済むところ、VS は候補5つ+確率分布+最終回答で 130〜300 tokens 使う。

- **多様性は 1.5〜2倍向上するが、トークンは 2〜6倍**
- naive_diverse はほぼ baseline と同コストで多様性もそこそこ上がるので、**コスパは naive のほうが良い**
- grok-3 が最も倍率が高い。今回の範囲では、モデルが賢くなるほど分布言語化が簡潔になるように見える

## 所感: どこで使うか

| 場面 | 推奨 |
|---|---|
| 創作・探索・シミュレーション（多様性が価値） | VS |
| 低遅延・コスト重視だけど脱テンプレしたい | naive_diverse |
| 事実QA・正解が1つのタスク | どちらも不要（多様性≒間違いのバリエーション） |

VS は「考えさせる足場」にトークンを使うので、レイテンシとコストが許容できるタスクで真価を発揮する。逆に、正解が決まっているタスクでは多様性を上げること自体がリスクになりうる。

## 注意点

- **N = 5 は統計的に弱い**。傾向を見るだけの雑実験
- **メトリクスが表面的**。文字レベルの n-gram 多様性は意味的多様性とは別
- **最終回答の抽出が不安定**。「【最終回答】」のフォーマットが揺れることがある
- **日本語のみ**。英語での結果は異なる可能性
